# Nomad Korea — Claude Code 가이드

## 프로젝트 개요

한국 디지털 노마드를 위한 플랫폼. Next.js 15 App Router 기반 홈페이지로, 도시 랭킹, 밋업, 코워킹 스페이스, 리뷰 기능을 제공한다. 모든 데이터는 `lib/mock-data.ts` 기반 Mock 데이터이며, 인증은 localStorage 기반 Mock 인증이다.

**스택**: Next.js 16 App Router · TypeScript · Tailwind CSS v4 · Base UI (`@base-ui/react`) · Supabase (인증 연동 준비 중)

> **주의**: UI 컴포넌트는 Radix UI가 아닌 `@base-ui/react`를 사용한다. `asChild` prop을 사용할 수 없다.

## 테스트 구조

### 단위/통합 테스트 — Vitest + Testing Library

```
__tests__/
├── components/    # 컴포넌트 단위 테스트
├── features/      # 기능별 통합 테스트
├── lib/           # 유틸리티 함수 테스트
├── providers/     # Context Provider 테스트
├── __mocks__/     # Next.js 모듈 mock (navigation, link, image 등)
└── setup.ts       # 전역 테스트 설정
```

```bash
npm test            # watch 모드
npm run test:run    # 단발 실행
npm run test:ui     # Vitest UI
```

### E2E 테스트 — Playwright

실제 브라우저에서 사용자 흐름을 검증한다. `npm run dev` 서버를 자동으로 띄운 뒤 테스트를 실행한다.

```bash
npm run test:e2e          # headless 실행
npm run test:e2e:ui       # Playwright UI 모드 (시각적 디버깅)
npm run test:e2e:debug    # 브라우저 열어서 step-by-step 디버깅
npm run test:e2e:report   # 마지막 실행 HTML 리포트 확인
```

#### 설계 원칙

**Page Object Model (POM)**
- 각 라우트별로 POM 클래스를 만든다. spec 파일에서 DOM 셀렉터를 직접 노출하지 않는다.
- 모든 POM은 `BasePage`를 상속한다. Navbar 이동·로그아웃 등 전 페이지 공통 액션은 `base.page.ts`에 둔다.
- 셀렉터는 `getByRole`, `getByTestId` 우선순위로 작성한다. CSS 클래스 셀렉터는 피한다.

**Fixture**
- 인증이 필요한 테스트(`profile.spec.ts` 등)는 `e2e/fixtures`의 `loggedInPage` fixture를 사용한다.
- `import { test, expect } from "../fixtures"` 로 가져온다.
- 인증이 필요 없는 테스트는 `@playwright/test`에서 직접 가져온다.

**테스트 계정**
- Mock 인증 기반이므로 실제 DB 불필요.
- `TEST_USER = { email: "test@test.com", password: "password123", nickname: "테스트유저" }`

**설정 파일**: `playwright.config.ts` (프로젝트 루트)
- `testDir`: `./e2e/tests`
- `baseURL`: `http://localhost:3000`
- `webServer`: `npm run dev` 자동 실행, 기존 서버 재사용(`reuseExistingServer: true`)
- 브라우저: Chromium 단일 프로젝트 (CI에서 필요 시 확장)
